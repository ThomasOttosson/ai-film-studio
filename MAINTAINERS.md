# MAINTAINERS.md

This document lists the responsibilities and expectations for project maintainers.

## Responsibilities

- Review pull requests in a timely manner.
- Keep documentation up to date.
- Maintain release quality.
- Respond to security reports.
- Help triage issues and discussions.

## Becoming a Maintainer

Prospective maintainers should:

1. Demonstrate consistent, high-quality contributions.
2. Follow the project's coding and review standards.
3. Be nominated by an existing maintainer.
4. Receive approval from the maintainer team.

## Inactive Maintainers

Maintainers who are inactive for an extended period may be moved to emeritus status after discussion with the team.

## Conflict Resolution

When maintainers disagree on significant technical decisions, document the alternatives, discuss them openly, and seek consensus before merging.
