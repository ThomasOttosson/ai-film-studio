# ARCHITECTURE_DECISIONS.md

This document tracks significant architectural decisions for the project.

## ADR-0001: Modular Architecture
**Status:** Accepted

The application is organized into modular features to simplify maintenance,
testing, and future expansion.

## ADR-0002: CI/CD Automation
**Status:** Accepted

Automated quality checks, security scanning, and release workflows are required
for all changes before production deployment.

## ADR-0003: Security by Default
**Status:** Accepted

Security reviews, dependency monitoring, least-privilege access, and secret
management are mandatory parts of the development lifecycle.

## Updating

Add new ADRs sequentially. Do not modify historical decisions; instead create a
new ADR that supersedes an older one when necessary.
