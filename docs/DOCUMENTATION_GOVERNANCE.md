# Documentation Governance

## Purpose

This document defines how documentation for AI Film Studio is created, reviewed, approved, versioned, and retired.

## Scope

This policy applies to:

- Architecture documentation
- Operational documentation
- Security documentation
- API documentation
- Runbooks
- Policies and standards
- User and developer guides

## Principles

- Documentation is treated as code.
- Documentation changes are reviewed alongside related implementation changes.
- Documents should have a clear owner.
- Outdated documentation should be updated or retired promptly.

## Document Metadata

Each maintained document should include:

- Title
- Purpose
- Scope
- Owner
- Review frequency
- Last review date

## Review Cadence

- Critical operational documents: every 6 months
- Security documentation: annually or after major security changes
- All other governance documents: annually

## Version Control

Documentation should:

- Be stored in version control
- Be reviewed through pull requests
- Reference related documents where appropriate
- Avoid duplicated guidance

## Retirement

Retired documents should be clearly marked or removed after confirming they are no longer referenced.
