# Support

This document explains how to get help with AI Film Studio.

## Before requesting help

Please check:

- Existing GitHub issues
- Project documentation
- `README.md`
- `docs/PRODUCTION_CHECKLIST.md`
- Recent release notes

## Bug reports

Use the GitHub bug report form and include:

- Clear reproduction steps
- Expected and actual behavior
- Browser, operating system, and deployment environment
- Relevant logs or screenshots
- The application version or commit SHA

Do not include passwords, access tokens, API keys, private media, or personal data.

## Feature requests

Use the GitHub feature request form. Describe the user problem, the proposed behavior, and acceptance criteria.

## Security vulnerabilities

Do not report security vulnerabilities in public issues.

Follow the private disclosure process described in `SECURITY.md`.

## Operational incidents

For production incidents:

1. Confirm the affected environment and user impact.
2. Capture timestamps, request IDs, and relevant logs.
3. Check recent deployments and infrastructure changes.
4. Apply the documented rollback process when necessary.
5. Record the incident and follow-up actions after recovery.

## Response expectations

Support is provided on a best-effort basis unless a separate service agreement exists. Response times are not guaranteed.
