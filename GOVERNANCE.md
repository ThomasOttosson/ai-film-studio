# AI Film Studio Governance

## Project Maintainers

Project maintainers are responsible for:

- Reviewing pull requests
- Managing releases
- Triaging issues
- Enforcing project standards
- Coordinating roadmap decisions

## Decision Making

- Minor changes may be approved by one maintainer.
- Significant architectural changes should be reviewed by at least two maintainers.
- Security-sensitive changes require additional review.

## Coding Standards

- All changes must pass CI.
- New features should include tests where practical.
- Public APIs should remain backwards compatible unless documented.

## Release Ownership

Each release should identify:

- Release manager
- Version
- Release date
- Deployment status
- Rollback owner

## Code of Conduct

Contributors are expected to follow the project's Code of Conduct and maintain a respectful, collaborative environment.
